Goal

HTML based template to create invoices by modifying HTML. Using that HTML to export a PDF by using a web browser's print dialog.

What it does

  • Automatically calculates invoice total and rows.
  • Each invoice references shared files to reuse styles, header, and footer.
  • Diff support between each invoice.

Usage

Steps to create a new invoice.
  1. Only for the first time using this, update invoiceFromTo.htm with your From & To information.
  2. Create a copy of invoice2022.8.example.htm and rename it to match the invoice date.
  3. Open the copied file in your text or html editor of choice.
  4. Update fields.
    1. invoice-number
    2. invoice-date
    3. rate-per-unit - Update this only if there is a change in your rate since your last invoice. This rate is per day. It is not shown in the PDF. It is used as a multiplier for the "units" in the line items beneath it.
    4. line items - Each line item is made up of 2 <span>s. First <span> is the description of the line item. The second line is the number of units used to complete it. Copy these 2 <span>s for as many line items as you need.

Architecture

3 files are referenced by each invoice. These 3 files create common styles and calculations on all invoices. The example invoice is invoice2022.8.example.htm. That is the file that should be copied and modified for each new invoice.
  • invoice2022.8.example.htm - File that should be copied, then modified, for each invoice.
  • invoiceFromTo.htm - File that should be updated only once. Contains your information and the information of who is being billed.
  • invoiceMath.js & invoicestrivr.css - Files that never need to be modified. Unless you'd like to change the appearance of the invoice.
